# Architecture of project

## Overview

My personal portfolio and blog, built with **Astro 5** and **Tailwind CSS 4**. It is a static site deployed automatically to GitHub Pages on every push to `main` via GitHub Actions.

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | Astro 5 |
| Styling | Tailwind CSS 4 (via `@tailwindcss/vite`) |
| Language | TypeScript |
| Package manager | pnpm |
| Deployment | GitHub Pages (GitHub Actions) |

## Directory Structure

```
src/
├── components/       # UI components (Atomic Design)
│   ├── atoms/        # Primitive elements
│   ├── molecules/    # Composed components
│   └── organisms/    # Complex sections
├── content/          # Content Collections (data layer)
│   ├── blog/         # Markdown blog posts
│   ├── projects/     # JSON project definitions
│   └── config.ts     # Collection schemas
├── layouts/          # Page shell templates
├── pages/            # File-based routing
├── styles/           # Global CSS
└── utils/            # Shared helper functions
```

## Component Architecture (Atomic Design)

Components are organized following the Atomic Design methodology:

```
atoms/
  Button.astro        # Variants: primary | secondary | outline | ghost | danger
  Heading.astro       # Semantic levels h1-h6 with size prop
  Text.astro          # Paragraph text with color and size variants
  Link.astro          # Anchor wrapper
  Badge.astro         # Category label chip
  Icon.astro          # SVG icon wrapper
  RemoteImage.astro   # External image with lazy loading

molecules/
  Card.astro          # Generic title + body card
  ProjectCard.astro   # Card with tech badges, category badge, and action links
  BlogPostCard.astro  # Card with image, date, author, and tags
  Navigation.astro    # Nav links with active state
  SocialLinks.astro   # Social media icon links
  TechIcons.astro     # Tech stack icon grid

organisms/
  Header.astro        # Site header with Navigation
  Footer.astro        # Site footer
```

## Layouts

Two layouts wrap all pages:

- **`Layout.astro`** — main shell used by all standard pages. Accepts `title` and `currentPath` props. Renders `Header` → `<slot />` → `Footer`.
- **`LayoutPost.astro`** — extends Layout for blog post pages, adds post-specific structure (title, date, author, tags).

## Data Layer (Content Collections)

Managed by Astro Content Collections, defined in `src/content/config.ts`.

### `posts` collection
Source: `src/content/blog/*.md`

```typescript
{
  title: string
  pubDate: Date
  description: string
  author: string
  image: { url: string; alt: string }
  tags: string[]
}
```

### `projects` collection
Source: `src/content/projects/*.json`

```typescript
{
  title: string
  description: string
  technologies: string[]
  githubUrl?: string
  demoUrl?: string
  projectCategory: 'Web' | 'Infrastructure Projects' | 'Security Tools'
  category: 'primary' | 'secondary' | 'success' | 'warning' | 'danger'
  image?: string          // path under /public/projects/
  featured: boolean       // shown on home page when true
  order: number           // ascending sort order
}
```

## Utility Functions

`src/utils/projects.ts` wraps `getCollection('projects')` with typed helpers:

| Function | Description |
|---|---|
| `getAllProjects()` | All projects sorted by `order` |
| `getFeaturedProjects()` | Projects with `featured: true`, sorted by `order` |
| `getProjectsByCategory(category)` | Filtered by `projectCategory`, sorted by `order` |
| `getProjectStats()` | Counts by category (`total`, `webApps`, `infrastructure`, `security`) |

## Routing

Astro uses file-based routing. All routes are statically generated at build time.

| URL | File |
|---|---|
| `/` | `src/pages/index.astro` |
| `/blog` | `src/pages/blog.astro` |
| `/posts/[slug]` | `src/pages/posts/[...slug].astro` |
| `/proyectos` | `src/pages/proyectos.astro` |
| `/sobre-mi` | `src/pages/sobre-mi.astro` |
| `/contacto` | `src/pages/contacto.astro` |
| `/tags` | `src/pages/tags/index.astro` |
| `/tags/[tag]` | `src/pages/tags/[tag].astro` |
| `/rss.xml` | Generated by `@astrojs/rss` |
| `/sitemap-*.xml` | Generated by `@astrojs/sitemap` |

## Deployment

GitHub Actions workflow (`.github/workflows/deploy.yml`) triggers on push to `main`:

1. **build** — uses `withastro/action@v3` to install dependencies, type-check, and build.
2. **deploy** — uses `actions/deploy-pages@v4` to publish to GitHub Pages.

The site is served at the custom domain configured in `CNAME` (`carlosmmairena.com`).
